## 簡単な紹介

`File Transfer Tools` 含む`FTS (File Transfer Server) `、、、、`FTC (File Transfer Client) `2つのコンポーネント、はい**軽量**としても**速い**としても**安全性**としても**多機能**Cross -deviceファイル送信スクリプト。

### 関数

1. ファイル転送

  - 単一のファイルまたはフォルダー全体を送信できます
  - 安全保証：暗号化された伝送（コンセンション - 接続層プロトコルを使用）、明示的な伝送
  - 特定の保証：ハッシュ値検証ファイルの一貫性を通じて、フォルダー内のすべてのファイルが正しく送信されているかどうかを判断します
  - 進行状況バーディスプレイ：リアルタイムディスプレイファイル伝送の進行状況、現在のネットワークレート、残りの伝送期間
  - 新しく名前が付けられたトランスミッションは、繰り返されるトランスミッションを避け、同じ名前の伝送をカバーする

2. コマンドラインは、リモコンでコマンドを簡単に実行し、sshと同様にリアルタイムで結果を返すことができます
3. サービスホストを自動的に見つけると、接続ホストを手動で指定することもできます
4. フォルダーの比較は、2つのフォルダーのファイルの情報、同じ違いなどを表示できます。
5. クライアントとサーバーのシステムシステムと情報のステータスと情報と情報を確認してください
6. REAL -TIME出力ログはコンソールとファイルへのログ
7. インターネット速度テスト

### 特性

1. 開始、実行、および応答の高速速度
2. ローカルエリアネットワークやパブリックネットワークなどのどのネットワーク環境でも使用できます。
3. マルチスレッド送信、高速トランスミッション速度は、実際の測定で1000Mbpsを超える帯域幅を実行できます。機器の制限により、より高い帯域幅はテストされていません
4. 記憶の職業は、ランタイム中は小さいです
5. つまり、オープンで、プロセスをオフにしてもプロセスは残りません

### 選択方法

1. より強力なファイル送信サービスが必要な場合は、FTPサーバーとクライアントを選択します（`FileZilla`としても`WinSCP`待って）
2. 安定したファイルの同期と共有が必要な場合は、使用することをお勧めします`Resilio Sync`としても`Syncthing`待って
3. ファイルを時々送信するだけで/上記のサービスのバックグラウンド保持が気に入らない場合、リソース職業/強力なサービスはありません/独自の機能をカスタマイズしたい場合は、選択してください`File Transfer Tools`

## インストールと操作

`FTS`占領ポート2023,2021、FTCはポート2022を占有します。その中で、ポート2023は次のように使用されます`FTS`TCPリスニングポート、2021、2022は、サーバーとクライアントの間のUDP送信インターフェイスとして。この記事の最後に詳細を確認できます。

### 実行可能プログラムをダウンロードします

1. 右側をクリックします`Release`
2. ダウンロード`File Transfer Tools.zip`
3. 解凍フォルダー、ダブルクリック`FTC.exe` また `FTS.exe` ただ走れ
4. または、端末でプログラムを実行して、たとえばプログラムパラメーターを使用します`.\FTC.exe [-h] [-t thread] [-host host] [-p]`

### Pythonインタープリターを使用して実行します

1. ソースコードをプロジェクトの場所にクローンします
2. 使用`pip install -r requirements.txt`すべての依存関係をインストールします
3. Pythonインタープリターを使用して、スクリプトを実行します

#### 練習方法

Windowsを例にとると、FTSとFTCSの実行中のコマンドをバッチファイルとして記述し、バッチファイルのディレクトリを環境変数に追加して、コマンドラインにコマンドラインを入力できるようにすることができます。`FTS`としても`FTC`プログラムを実行するために、デフォルトと最もシンプルなコマンドを使用しましょう。

たとえば、次のコマンドをファイルに書き込むことができます`FTS.bat`真ん中


次のコマンドをファイルに書き込みます`FTC.bat`真ん中

```powershell
@echo off
"The dir of your Python interpreter"\Scripts\python.exe "The dir of your project"\FTC.py %1 %2 %3 %4 %5 %6
```

次に、バッチフォルダを環境変数に追加し、最後に端末に次のコマンドを入力して、コードをすばやく実行します


## 使用法

### FTC

FTCは、ファイルと命令を送信するためのファイルを送信するファイル、命令を送信する命令です。

```
usage: FTC.py [-h] [-t thread] [-host host] [-p]

File Transfer Client, used to SEND files.

optional arguments:
  -h, --help       show this help message and exit
  -t thread        threading number (default: 3)
  -host host       destination hostname or ip address
  -p, --plaintext  Use plaintext transfer (default: use ssl)
```

#### パラメーターの説明

`-t`：スレッドの数を指定します。デフォルトは3つのスレッドです。

`-host`：受信側のホストを明示的に指定します（ホスト名またはIPアドレスを使用）。このオプションを使用していない場合、クライアントは自動的に見つけます**同じサブネット**サーバー

`-p`：協力します`-host` 2つの当事者は自動的に情報を交換するため、通常は指定する必要はありません。2つの当事者が正常に接続できない場合にのみ、明らかに指定する必要があります。

#### コマンド指示

通常の接続の後、指示を入力します

1. ファイル（クリップ）パスを入力してから、送信ファイルを実行します
2. 入力`sysinfo`、両当事者のシステム情報を表示します
3. 入力`speedtest n`、ネットワークの速度をテストします。**ネット**中央では、1 gb = 1000 mb = 1000000 kb。
4. 入力`compare local_dir dest_dir`フォルダーとサーバーフォルダーの違いを比較しましょう。
5. 他のコンテンツを入力するときは、指示として実行し、結果をリアルタイムで返します。

### fts

`FTS`これは、ファイルの受信および保存に使用され、クライアントからの命令を実行するために使用されるサーバー側のファイルです。

```
usage: FTS.py [-h] [-d base_dir] [-p] [--avoid]

File Transfer Server, used to RECEIVE files.

optional arguments:
  -h, --help            show this help message and exit
  -d base_dir, --dest base_dir
                        File storage location (default: C:\Users\admin\Desktop)
  -p, --plaintext       Use plaintext transfer (default: use ssl)
  --avoid               Do not continue the transfer when the file name is repeated.
```

#### パラメーターの説明

`-d, --dest`：ファイルのストレージ位置を指定します。指定されていない場合は、現在のユーザーに保存されます。**デスクトップ**次に、

`-p`：明示的な伝送を指定し、デフォルトでSSL暗号化伝送を使用します。現在署名証明書がない場合は、明示的な送信を指定してください。**セキュリティを確保するために、独自の署名証明書を使用してください。**

`--avoid`：開く時点で、ディレクトリに同じ名前のファイルが既にある場合、2つのケースに2つのケースがあります。**防ぐ**このファイルの送信、それ以外の場合は受信され、**上書きします**このファイル;この関数は、主に中断された後に多数のファイルを送信するために使用されます。**慎重に使用してください**開いていない場合、ファイルに名前が付けられている場合`a.txt`その後、送信されたファイルは次のとおりです`a (1).txt`としても`a (2).txt`順番に名前が付けられました。
